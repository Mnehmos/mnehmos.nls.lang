# Atomizer Stress Tests

This directory is for finding weaknesses in the Python -> NLS -> Python loop.

## Repository Roundtrip

Use the repo roundtrip runner to mirror a Python repository into NLS artifacts and regenerated Python:

```bash
python stress-test/atomize_repo_roundtrip.py <source-repo> <output-dir> --max-files 50
```

What it writes under `<output-dir>`:

- mirrored `.nl` files generated by the atomizer
- mirrored `.roundtrip.py` files generated by compiling those `.nl` files back to Python
- `roundtrip-report.json` summarizing which files passed and which failed

This is the fastest way to build an "intent-code" version of an existing Python codebase without changing your original repo naming. Point the tool at any destination directory that matches the naming convention you want for the converted repo.

## What Failures Mean

- `stage: ok` - atomize, parse, emit, and any supplied behavioral checks matched
- `stage: behavior` - the compiler loop ran, but behavior changed
- `stage: parse` - atomized NLS could not be parsed back into the compiler
- `stage: roundtrip` - NLS parsed, but regenerated Python failed to emit or execute

## Recommended First Pass

Start with a few representative repos and cap the scan size:

```bash
python stress-test/atomize_repo_roundtrip.py ../some-python-repo ../some-python-repo-intent --max-files 25
```

Then inspect `roundtrip-report.json` for recurring weak spots such as:

- variadic functions (`*args`, `**kwargs`)
- decorators and class-bound methods
- context managers
- exception-heavy functions
- complex type annotations
